# AGENTS

- La base primordial del trabajo es `specs/constitution.md`; siempre leerla y seguir el workflow que indica antes de proponer, planificar o implementar.
- Si el usuario indica el nombre de un spec, trabajar en base a `specs/<nombre>/spec.md`, luego revisar `plan.md` y `test-cases.md` del mismo directorio segun el flujo de la constitucion.
- Si el spec no existe o no cumple los gates de la constitucion, detener el avance de implementacion y primero corregir o pedir definicion del spec.
